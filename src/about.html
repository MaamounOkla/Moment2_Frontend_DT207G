Beskrivning av lösningen



Författare: Maamoun Okla


Kurs: DT207G Backend-baserad Webbutveckling


Termin: VT24




1. Syfte

En webbapplikation för att lagra arbetslivserfaerenheter som består av två mindre projekt, ett för en frontend-webbplats (den här webbplatsen) och det andra för ett API (den REST-baserade webbtjänsten) som är kopplat till en relationsbaserad databasserver (MySQL) för lagring. Webbplatsen konsumerar webbtjänsten med hjälp av Fetch API och med det så kallade CRUD (Create, Read, Update och Delete) för att hantera data i databasen. Syftet är alltså att förstå hur moderna plattformsberoende applikationer fungerar och kunna implementera detta i nästkommande projekt.



2. Konstruktion och utvecklingsmiljö

Teknik som användes för att bygga webbapplikationen:
 

  • Visual Studio Code eller ett annat IDE.
    •  
  • Webbläsare, till exempel Chrome eller FireFox.
    •  
  • NodeJS
    •  
  • Xampp

    • Paketen som användes i backend för att bygga API:et för denna webbapplikation:
     
  • Express för att skapa servern och ansluta till en relationsdatabas för att lagra kursinformation.
    •  
  • CORS för att tillåta Fetch API från andra domäner.
    •  
  • MySQL som relationsdatabas, andra relationsdatabaser som till exempel SQLite eller MariaDB fungerade också bra.
    •  
  • phpMyAdmin som ett gränssnitt för att underlätta databashantering under utvecklingsfasen. Detta körs med hjälp av Xampp.
    •  
  • Nodemon för att underlätta port-hantering och uppdatering av applikationen under utvecklingsfasen.
    •  
  • Dotenv för att hantera .env-filer och deras globala variabler.



    • 3. Slutsats

    Jag tycker att kunna bygga en fullständlig applikation (det så kallade fullstack applikation) från databasen till API:et till frontenden samt att kunna konsumera webbtjänsten som man byggde är mycket fördelaktigt och ger full kontroll över nästan allting i webbapplikationen. Man kan se hur kraftfull webbapplikationen är och utvecklingsmöjligheterna blir oändliga. Det öppnar dörren till en hel ny värld inom IT där man kan bygga moderna applikationer och ha kontroll över sin data. Detta är en väldigt kraftfull verktyg som stödjer webbutvecklaren med sitt jobb och löser många affärsbehov.

    Det finns många fördelar med detta som till exempel:
     

  • Full kontroll över applikationen och datahanteringen.
    •  
  • Möjlighet att anpassa webbapplikationen som man vill genom att strukturera API:et som man vill.
    •  
  • Flexibiliteten att anpassa efter olika situationer och olika behov och krav.
    •  
  • Öppnar dörren till ett mycket lärorikt område inom webbutvecklingen.

    • Det finns dock några nackdelar med detta:
     
  • Att bygga ett eget API och själv bestämma reglerna är mycket mer komplext än att använda och konsumera ett färdigt webbtjänst; det kräver dessutom mycket mer tid.
    •  
  • Större chans att webbapplikationen kraschar och ingenting renderas för användaren.
    •  
  • Felmeddelanden hanteras inte själva utan man måste vara noggrann med att ge rätt felmeddelande, och detta kräver bra analysförmåga för att kunna beskriva till användaren vad som har hänt vid fel.
